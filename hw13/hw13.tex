\noindent
\textbf{\large 13.1}

假设一个块的大小为 4KB，则创建 /home/OS24/fs03.pdf 时，需要修改位图块、创建 fs03.pdf 的 inode 块、修改目录块、修改 /home/OS24 的 inode 块、添加 3 个数据块并写入、再次修改 fs03.pdf 的 inode 块，共需要修改 7 个块。

下面列举出任意时刻宕机后出现不一致的情况：

\begin{itemize}
  \item 在修改位图块后宕机，此时位图块已经被修改，但是其他块没有被修改，会导致位图块中的数据块被分配但是其他块没有被修改。
  \item 在创建 fs03.pdf 的 inode 块后宕机，此时 fs03.pdf 的 inode 块已经被创建，但是并没有修改目录块，会导致 fs03.pdf 的 inode 块被创建但是目录块中并没有 fs03.pdf 的记录。
  \item 在修改目录块后宕机，此时目录块已经被修改，但是并没有修改 /home/OS24 的 inode 块，会导致目录块中的 fs03.pdf 记录被修改但是 /home/OS24 的 inode 块中并没有 fs03.pdf 的记录。
  \item 在修改 /home/OS24 的 inode 块后宕机，此时 /home/OS24 的 inode 块已经被修改，但是数据块并没有被修改，导致元数据与数据不一致。
  \item 在添加数据块并写入后宕机，此时数据块已经被写入，但是 fs03.pdf 的 inode 块并没有被修改，导致数据块中的数据没有被记录。
\end{itemize}

\textbf{\large 13.2}

\begin{enumerate}[label=(\arabic*)]
  \item 若在日志完成 TxE 之前宕机，则磁盘内日志不完整，什么也不做，文件A的内容仍为旧的内容；若在日志完成 TxE 之后宕机，由于磁盘内日志完整，则按照日志中的内容进行恢复，文件A的内容为新的内容。
  \item 若在日志写 TxB 前宕机，则元数据不变，新写入的数据视为垃圾块，文件A的内容仍为旧的内容；若在日志写 TxB 后写 TxE 前宕机，元数据日志不完整，则元数据不变，新写入的数据视为垃圾块，文件A的内容仍为旧的内容；若在日志写 TxE 后宕机，由于日志完整，按照日志中的内容进行恢复，元数据更新，文件A的内容为新的内容。
\end{enumerate}

\noindent
\textbf{\large 13.3}

\begin{enumerate}[label=(\arabic*)]
  \item 每个imap可以索引到$\dfrac{4000}{4} = 1024$个inode，因此需要$\lceil\dfrac{2000000}{1024}\rceil = 1954$个imap块。
  \item 每个CR可以索引到$\dfrac{4000}{4} = 1024$个imap块，因此需要$\lceil\dfrac{1954}{1024}\rceil = 2$个CR块。
  \item ino=356302的inode在第$\lceil\dfrac{356302}{1024}\rceil = 348$个imap块的第$356302-1024\times 347 = 974$个位置。因此只需读取第一个CR块的\texttt{CR[347][973]}即可找到ino=356302的inode。
\end{enumerate}

\noindent
\textbf{\large 13.4}

\begin{enumerate}[label=(\arabic*)]
  \item 直接指针的索引范围为4KB，一级间接指针的索引范围为$4 \times \dfrac{4 \times 1024}{4} = 4MB$，二级间接指针的索引范围为4GB，三级间接指针的索引范围为4TB。故文件foo使用了直接指针、一级间接指针、二级间接指针。
  \item 文件foo总共占用的磁盘块数为$\dfrac{10 \times 1024}{4} = 2560$块，则需要写入的为foo的最后一块。写流程为：输入数据写数据块、写inode、更新imap，共3次I/O操作。
  \item 同样写最后一块，但是采用FFS的写流程为：写数据块、写inode、更新bitmap，共3次I/O操作。
  \item 如果是日志文件系统，则增加写TxB、写修改操作、写TxE，共6次I/O操作。
\end{enumerate}
