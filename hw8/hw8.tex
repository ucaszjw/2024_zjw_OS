\textbf{8.2}

\begin{enumerate}[label=(\arabic*)]
    \item 页大小即为$2^{12} = 4$KB。
    \item 共能索引$2^{8+6+6} = 2^{20}$个页。
    \item 由于页大小为4KB，故代码段占据的页数为$\lceil \dfrac{128}{4} \rceil = 32$页，数据段占据的页数为$\lceil \dfrac{66}{4} \rceil = 17$页，栈段占据的页数为$\lceil \dfrac{8}{4} \rceil = 2$页。所以最少占用的物理页框数为$32+17+2=51$，最多占用的极端情况为每个物理页框只使用1字节，共占用$128K + 66K + 8K = 202K = 101 \times 2^{11}$个物理页框。
    \item 由于页大小不变，故程序需要占用的物理页框数量不变，仍为最少占用的物理页框数为$51$，最多占用的极端情况为$101 \times 2^{11}$个物理页框。
\end{enumerate}

\noindent
\textbf{8.3}

可设TLB命中率为$h$，则TLB未命中率为$1-h$。

有公式：
\[
    \text{虚实地址映射平均延迟} = h \times \text{TLB查询时间} + (1-h) \times (\text{TLB查询时间} + \text{页表查询时间} + \text{TLB填充时间})
\]

若使得虚实地址映射平均延迟为40ns，则有：

\begin{align*}
    40 &= 5h + 130(1-h) \\
     h &= 0.72
\end{align*}

故TLB命中率至少为72\%。

若使得虚实地址映射平均延迟为20ns，则有：

\begin{align*}
    20 &= 5h + 130(1-h) \\
     h &= 0.88
\end{align*}

故TLB命中率至少为88\%。

\textbf{8.4}

\begin{enumerate}[label=(\arabic*)]
    \item 由于虚地址空间为$2^{32}$，页大小为4KB，故页表项数为$2^{32-12} = 2^{20}$。
    \item 要使得每次进入循环都触发TLB未命中，即每次进入循环都要访问一个新的页表。而一个页表内可容纳$2^{12-2} = 2^{10}$个数组元素，只需步长$M > 2^{10}$即可。对于数组元素个数，只需要$N \geq M$即可。
    \item 要使TLB命中尽可能多，即每次进入循环都尽量访问相同的页表项，只需步长$M = 1$即可。对于数组元素个数，只需要$N = 2^{10}$即可。
\end{enumerate}
